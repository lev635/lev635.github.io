Github Pagesでwebページを作成

最終更新:

概要

Github Pagesを用いて自分のwebページを作成するまでの手順をまとめた。

手順

リポジトリの作成

webページの中身を入れるリポジトリを作成する。リポジトリの名前は何でも良いが、これによりURLが決まる。 公式サイト ではリポジトリの可視性を選択できるかのように書かれているが、執筆時点では無料プランの場合Publicしか選択できない点に注意。

webサイトを作る際にはローカル環境でテストすることが多いだろうから、リポジトリをcloneしておく。

webページの作成

公開したいwebページを作成し、mainブランチにpushする。webページはまだ出来ていないが動作を確認したいという場合は、 以下のような適当なHTMLファイルをpushしておく。ファイル名はindex.htmlとしておく。 


<!DOCTYPE html>
<html>
<head>
    <title>test</title>
</head>
<body>
    <h1>test</h1>
</body>
</html>

Github Pagesの設定

先程作成したリポジトリにおいてsettings/pagesを開く。 Build and deploymentの設定を変更する。Sourceの項をDeploy from a branchに、 Branchの項をmainとする。

しばらく待つと "Your site is live at ..." と出る。リンクをクリックすると自分のページが表示される。

注意点

サイズ制限

Github Pagesではページのサイズは 1GB を超えてはならない。つまり、大きな画像を貼り付けるのは控えるべきだし、 HTMLやCSSの使いまわせる部分は積極的に使いまわすべきだということになる。

静的サイト

Github Pagesは静的サイトである。そのため、ユーザーに応じて表示するものを変更することなどは出来ない。
